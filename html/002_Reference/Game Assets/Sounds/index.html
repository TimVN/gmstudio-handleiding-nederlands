Sound

GameMaker:Studio has two separate sound engines that you can use in your games - The Legacy Sound and the GM:S Audio. This section explains the differences between the two as well as providing all the necessary functions to work with them.

GameMaker:Studio has a re-vamped audio engine that is based on the *.ogg sound format. You must still load your sounds into the IDE as either *.wav or *.mp3 but the runner will convert these when your finished game is compiled. In this way GameMaker:Studio keeps the final file size of your game to a minimum while ensuring maximum quality. The added benefit of the new GM:S Audio engine is that you now have access to full 3D sound with the ability to place a "listener" for all sound as well as different sound "emitters" to create some stunning special audio effects!

Please note that you cannot have both the new audio engine and the legacy engine active at the same time. For old games created with other versions of GameMaker or for those games started before the new audio engine was added, the Legacy Audio will be used as default, while for all new games GameMaker:Studio will default to the new audio engine. This behaviour can be changed from the Global Game Settings - General Tab.

The legacy functions are still valid and available due to the fact that certain browsers for HTML5 games do not support the newer sound system. This means that you may wish to check in-game which sound system is actually being used and change the functions being used as appropriate. To that end there is a special function available:

  1. audio_system

Basic Audio Functions

The following functions are for dealing with audio in the most straightforward and simple way possible, avoiding the use of emitters and permitting the user to generate sounds and play music easily as these sounds are always generated at the listener position (see below for more details on the listener) and so are generally not affected by any changes to the listener:

  1. audio_exists
  2. audio_get_type
  3. audio_play_music
  4. audio_pause_music
  5. audio_stop_music
  6. audio_play_sound
  7. audio_play_sound_at
  8. audio_pause_sound
  9. audio_pause_all
  10. audio_resume_sound
  11. audio_resume_music
  12. audio_resume_all
  13. audio_stop_sound
  14. audio_stop_all
  15. audio_is_playing
  16. audio_music_is_playing

Advanced Audio Functions

The following functions are designed to give more control over how the audio engine works, and how the sounds played through it will be "heard" by the listener. As such it is recommended that you have a good working knowledge of how the rest of the GameMaker:Studio audio engine works before using any of the following functions:

  1. audio_sound_length
  2. audio_sound_pitch
  3. audio_falloff_set_model
  4. audio_music_gain
  5. audio_sound_gain
  6. audio_master_gain
  7. audio_channel_num

Emitters

Audio emitters are provided to increase the flexibilty of the GameMaker:Studio audio engine, and they permit you to add real time effects to your audio assets, like pitch and doppler variations, as well as the ability to position your sounds within the 3D audio space and give them realistic motion effects. All these functions are affected by the position of the listener within the audio environment and so will require that you use the provided functions for changing the listener position, velocity and orientation too.

NOTE : These functions can only be used with sounds that are flagged as "Normal Sound" on import into GameMaker:Studio and will not work with those flagged as "Background Music".

The following functions exist to deal with audio emitters:

  1. audio_emitter_create
  2. audio_emitter_exists
  3. audio_emitter_position
  4. audio_emitter_velocity
  5. audio_emitter_falloff
  6. audio_emitter_gain
  7. audio_emitter_pitch
  8. audio_emitter_free
  9. audio_play_sound_on

The Listener

When dealing with complex audio situations it can be useful to define where in the audio environment the listener should be situated. When working with the simple audio functions, it is not normally necessary for the listener to be changed in any way, however when using emitters to generate realistic audio within the 3D audio space, it is important that the listener be changed and moved correctly, or else the sound will be in the wrong position for the person playing your game.

GameMaker:Studio provides the following listener functions:

  1. audio_listener_position
  2. audio_listener_velocity
  3. audio_listener_orientation
