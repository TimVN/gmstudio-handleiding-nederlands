iap_status

Allows you to track the status of the in app purchasing system.

Syntax :

iap_status();


Returns : Real


Description

This function allows you you to track the status of the GameMaker:Studio in app purchasing system and returns a ds_map with the following two key/value pairs:

  • “status” - This key indicates the overall status of the in app purchase system and can return the following values:
    1. "Not Enabled"
    2. "Not Available"
    3. "Loading"
    4. "Available"
    5. "Processing Order"
    6. "Error" (This means that something went wrong with the transaction itself.)
    7. "Failed" (This means that the attempt to retrieve the content_url went wrong.)
  • “purchases” - If the set of available purchases has been retrieved correctly, the value of this key is a ds_list with each entry being a ds_map containing all the information for each available purchase (see iap_activate for more information on these maps) as well as an additional key entry marked "status". This extra “status” key hold a value that indicates the purchase status of the item, and can be one of the following:
    1. "Purchased"
    2. "Available"
    3. "Unavailable"
    Please note that for a 100% correct check of the purchase you should use the function iap_is_purchased.

NOTE: The ds_map returned is created by this function and maintened by GameMaker:Studio for all further calls. This means that you should not remove it from memory.


Example :

status_map = iap_status();

The above code requests the purchase status for the current game.
